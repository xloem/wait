Why Am I Talking: In Pursuit of Radio Silence

Radio hacking is real. You can buy a $20 tv-tuner and use open source software to tune into your neighbor's display and see what they're doing without even knowing where they are, by reconstructing it from signal leakage patterns that repeat at the refresh rate. These wide technologies are not common because people like having power. We need to understand these things better, and have the freedom to protect against them, because some are using them to control the lives of others.

Data Format for Raw RTL-SDR Dumps
  • First four bytes: uint32_t size of device structure
  • A stream of one of the following:
    • 1 byte LOG_DATA=00: A block of I/Q data received from the device. 4 bytes uint32_t data length, and then that many bytes of data. 1 byte I, 1 byte Q, repeat. Data is 0-255 unsigned, centered around 127.
    • 1 byte LOG_TIME=01: The current time from epoch follows as a struct timespec. This will be 4 bytes seconds, then 4 bytes nanoseconds.
    • 1 byte LOG_STRUCT=02: A change in device state. The current device state follows as a struct rtlsdr_dev. This is also a rtlsdr_dev_t which can be passed into the librtlsdr c functions and queried. The length of this data is the first 4 bytes in the file. I think the current tuned frequency is in MHz as a 4 byte unsigned value 128 bytes into the structure.
